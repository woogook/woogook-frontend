# Commit Workflow

## 목적

- 각 commit을 사람이 읽고 정리하기 쉬운 형태로 유지한다.
- 각 commit이 squash 판단에 필요한 최소 정보를 제목과 본문에 남기도록 한다.

## 입력

- 현재 branch
- 변경 요약
- 테스트 및 검증 결과
- 기준 GitHub Issue

## 실행 순서

1. commit 제목은 기본적으로 `type(scope): 한글 요약` 형식을 사용한다.
2. scope가 불필요하면 `type: 한글 요약`으로 단순화할 수 있다.
3. commit 제목의 `type` 또는 `type(scope)`는 영어 소문자로 유지하고, 요약은 한글을 기본으로 작성한다.
4. 영어가 더 정확하거나 자연스러운 경우에만 괄호로 병기한다.
5. non-trivial 변경이면 commit 본문을 함께 작성하고, 본문도 한글을 기본으로 작성한다.
6. commit 본문에는 아래 최소 항목을 짧게 남긴다.
   - 배경
   - 변경 내용
   - 검증
   - 필요 시 문서 영향 또는 후속 작업

## 완료 조건

- 제목만 보고 변경 의도를 파악할 수 있다.
- 본문이 필요한 commit에는 배경, 변경 내용, 검증이 남아 있다.
