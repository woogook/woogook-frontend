# AGENTS.md

이 문서는 `woogook-frontend` 프로젝트에서 사용하는 로컬 전용 운영 메모다.
프로젝트 전역에 적용되는 기본 작업 지침을 담고 있으며, 하위 디렉터리에 별도 `AGENTS.md`가 있으면 더 구체적인 문서가 우선한다.

## 적용 범위

- 이 문서는 Agent가 코드 또는 스크립트 파일을 수정할 때 기본 규칙으로 적용한다.
- 문서만 수정하거나 조사, 리뷰, 데이터 점검만 수행한 작업에는 작업 성격에 맞게 필요한 규칙만 적용한다.

## 작업 지침

- 필요 시 `Subagents`를 생성해서 효율적으로 작업한다.
- `영어`는 꼭 필요한 경우에만 사용하고, 그 밖의 내용은 `한글`로 작성한다.
- `디렉터리명`, `파일명`, `변수명` 등을 정할 때는 `glossary.md`의 내용을 먼저 참고하고, 가능한 한 그 표기를 따른다.
- 코드 또는 스크립트 파일을 수정한 뒤에는 관련 문서(`README.md`, `glossary.md`, `docstring` 등)를 함께 점검한다.
- 수정 대상 `README.md`는 변경 범위, 사용자 영향도, 운영 절차 변경 여부를 기준으로 Agent가 적절하게 판단한다.
- `docstring`은 아래 규칙에 따라 작성한다.
  - `docstring`이 있어야 하는 함수, 클래스, 모듈, 스크립트 등에 `docstring`이 없으면 새로 작성한다.
  - 이미 `docstring`이 있으면 동작 설명, 입력/출력, 부작용, 제약이 바뀌는 경우에만 수정한다.
  - 단순 리팩터링, 변수명 변경, 포맷 정리처럼 외부 동작 설명이 바뀌지 않으면 `docstring`은 수정하지 않아도 된다.

## 구체적인 작업 내용 기록

- Agent가 수행한 구체적인 작업 내용을 `<project_root>/tmp/adr/<yymmdd>` 디렉터리에 `markdown` 문서로 기록한다.
- 파일명은 `yymmdd-<sequence>-<주제>.md` 형식을 사용한다.
- 기록 문서는 아래 구조를 권장한다.
  - 제목
  - 배경
  - 변경 사항
  - 검증
  - 후속 메모
  - 기타
- 마지막 `기타` 섹션은 반드시 아래 내용을 포함한다.
  - git
    - git convention을 준수한 브랜치 이름 후보 3개
    - git convention을 준수한 구체적인 커밋 메시지 (prefix 포함)
    - git convention을 준수한 구체적인 PR 본문 내용 (title 포함)
