# 배경

- `npm audit`에서 `next` 직접 의존성에 대해 high severity vulnerability 1건이 보고됐다.
- `package.json`이 `next`를 `16.1.6`으로 정확히 고정하고 있어 `npm audit fix`만으로는 취약점 해소 버전까지 올라가지 않았다.

# 변경 사항

- `package.json`에서 `next`와 `eslint-config-next`를 `16.2.3`으로 올렸다.
- `npm install`로 `package-lock.json`을 재생성해 `@next/*` 연관 패키지와 lockfile 항목을 동기화했다.

# 비채택안

- `npm audit fix --force`: 직접 의존성의 고정 버전을 자동으로 바꾸는 대신 더 넓은 범위의 예기치 않은 업데이트를 유발할 수 있어 채택하지 않았다.
- 취약점 무시: `next < 16.2.3` 범위의 high advisory가 남아 있어 수용하지 않았다.

# 검증

- `npm audit --json`: high 1건에서 0건으로 해소됨을 확인했다.
- `npm run lint`: 실패. `.worktrees/.../.next` 생성 산출물까지 lint 대상에 포함되어 대량 오류가 발생했다.
- `npm run build`: 실패. 현재 실행 환경 Node.js `18.20.8`에서 Next.js가 요구하는 `>=20.9.0` 조건을 충족하지 못했다.

# 후속 메모

- 로컬 검증을 완전히 통과시키려면 Node.js를 `20.9.0` 이상으로 맞춘 뒤 다시 `npm run build`를 확인해야 한다.
- 별도 작업에서 ESLint 대상 경로에 `.worktrees`와 `.next` 생성물이 섞이지 않도록 ignore 구성을 점검할 필요가 있다.
