# Node 실행 환경 정리

## 배경

- `npm run dev` 실행 시 현재 로컬 Node.js 버전이 `v18.20.8`로 잡혀 `Next.js 16.1.6`의 최소 요구사항 `>=20.9.0`을 만족하지 못했다.
- 저장소 루트에는 `.nvmrc`나 `package.json` `engines` 설정이 없어, 필요한 Node 버전을 실행 전에 알기 어려웠다.
- `README.md` 실행 절차도 버전 전환 없이 `npm install`, `npm run dev`만 안내하고 있어 같은 문제가 반복될 수 있었다.

## 변경 사항

- 루트에 `.nvmrc`를 추가해 `Node.js 22` 계열을 기본 실행 버전으로 고정했다.
- `package.json`에 `engines.node: >=20.9.0`를 추가해 런타임 최소 요구사항을 명시했다.
- `README.md` 실행 절차는 팀 공용 문서 성격을 고려해 `Node.js 20.9.0 이상`이라는 최소 요구사항 중심으로 정리하고, 설치 절차는 특정 버전명이 아닌 `.nvmrc` 기준 `nvm install` 흐름으로 안내했다.
- 현재 워크트리에 있던 `.gitignore`, `package-lock.json` 변경도 함께 반영 대상으로 묶었다.

## 비채택안과 그 이유

- `Node.js 20.20.1` 같은 특정 패치 버전을 `.nvmrc`에 고정하는 방안:
  현재 작업 환경에는 해당 버전이 설치돼 있지 않았고, `Next.js 16`은 `>=20.9.0`이면 동작하므로 이미 설치된 `Node.js 22` 계열을 기본값으로 두는 편이 즉시 실행 가능성과 운영 단순성이 높았다.
- 문서만 수정하고 버전 파일은 추가하지 않는 방안:
  터미널에서 자동 감지되는 기준점이 남지 않아 실수 가능성이 계속 남는다. 재발 방지 효과가 약해 채택하지 않았다.

## 검증

- `nvm use` 후 `node -v` 결과가 `v22.21.0`인지 확인했다.
- `npm run lint`는 새 오류 없이 기존 경고 1건(`src/app/components/CandidateCards.tsx`의 `@next/next/no-img-element`)만 남았다.
- `npm run build`는 `2026_data/sample_ballot_response_*.json` 누락과 Google Fonts fetch 실패로 중단됐다. 전자는 기존 로컬 데이터 의존성 문제이고, 후자는 현재 Codex 샌드박스의 네트워크 제한 영향이다.

## 후속 메모

- 팀 공통 런타임을 더 엄격히 맞출 필요가 생기면 `.nvmrc`를 특정 패치 버전으로 조정하거나 CI `setup-node` 버전과 일치시키는 방안을 다시 검토할 수 있다.
- `npm run build`를 로컬에서 완전히 통과시키려면 `2026_data/` 샘플 파일 준비와 네트워크가 가능한 환경에서의 폰트 fetch 확인이 필요하다.

## 기타

- 현재 GitHub CLI `gh` 인증이 무효여서 PR 생성 자동화에는 별도 인증 갱신이 필요할 수 있다.
