# 공통 LLM 진입 문서

## 목적

- 공통 workflow와 GitHub helper를 어떤 시점에 읽어야 하는지 정한다.
- `프로젝트 실행`으로 분기된 요청에서 어떤 문서를 먼저 열어야 하는지 고정한다.

## 기본 진입 순서

1. `AGENTS.md`
2. 이 문서
3. `프로젝트 실행`이면 `docs/common/canonical/llm-workflow-harness.md`
4. 현재 단계에 맞는 `docs/common/codex/workflows/*.md`
5. 필요할 때 `docs/common/codex/guides/*.md`

## 적용 범위

- `common` 도메인 작업
- 도메인 분기 전 공통 판단이 필요한 요청

- `docs/common/canonical/llm-workflow-harness.md`
